# 翻訳品質問題の修正

`/check-quality` で検出された問題を修正する。

## 引数

$ARGUMENTS

引数が空の場合、以下を対話で確認する：

1. **対象法律**: 例 `minpou`, `keihou`（全法律は非推奨、範囲を絞ること）
2. **修正する問題の種類**:
   - `merchant` → 商人表現の置き換え（投資→力を注ぐ、利益→メリット等）
   - `male` → 男性表現の除去（わい、わいら、おんどれ等）
   - `short` → 短すぎる翻訳・解説の加筆
   - `missing` → 翻訳・解説なしの条文を新規翻訳
   - `examples` → 例え話のない解説に具体例を追加
   - `all` → 全種類

## 修正の方針

### 機械的に置換可能な問題（merchant, male）

- 対象条文を検索
- 置換パターンを提示（例：「投資」→「力を注ぐ」）
- ユーザー確認後に一括置換
- 置換後に文脈が自然か確認

### 人間的判断が必要な問題（short, missing, examples）

- 対象条文をリスト化
- 1条ずつ（またはバッチで）翻訳・加筆
- `.claude/guides/translation-style-guide.md` のスタイルに従う
- 品質基準：commentaryOsaka 300文字以上、具体例必須

## 修正後の確認

修正完了後、対象法律に対して品質チェックスクリプトを再実行し、問題が解消されたことを確認する。

## 対になるスキル

問題の検出には `/check-quality` を使用。
