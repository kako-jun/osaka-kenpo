# 条文の大阪弁翻訳

指定された条文のosakaText・commentary・commentaryOsakaを生成し、YAMLファイルに書き込む。

## 引数

$ARGUMENTS

引数が空の場合、以下を対話で確認する：

1. **対象法律のパス**: 例 `jp/minpou`, `world_hist/magna_carta`
2. **条文範囲**: 例 `1-50`, `全部`, `282`（単一条文）
3. **翻訳対象フィールド**: 全部 or 特定フィールドのみ（osakaText / commentary / commentaryOsaka）

## 必ず読み込むガイド

翻訳開始前に以下を読み込んでスタイルを把握すること：

- `.claude/guides/translation-style-guide.md`（翻訳スタイルガイド）

## 翻訳するフィールド

### osakaText（大阪弁訳）

- 原文の語尾・語調・語彙のみ変換（意味を変えない）
- 女性的な関西弁（春日歩風）
- 語尾バリエーションを意識

### commentary（標準日本語の解説）

- 条文の意味を分かりやすく解説
- 3段落以上、300文字以上
- 具体例を含める（「例えば」で始まる段落）

### commentaryOsaka（大阪弁の解説）

- commentaryの内容を大阪弁化するのではなく、独立した大阪弁の解説を書く
- 3段落以上、300文字以上
- 具体例必須（「例えばな、」で始まり、AさんBさん等の登場人物を使用）
- 商人表現禁止（投資、利益、儲け、商売、コスト等）
- 男性表現禁止（わい、わいら、おんどれ等）

## 品質基準

翻訳後、各条文が以下を満たしているか確認する：

- [ ] 原文の法的意味を完全保持
- [ ] 女性的な関西弁で統一
- [ ] 語尾が多様（同じ語尾の3回以上連続なし）
- [ ] 「知らんけど」は法律全体で1-2回まで
- [ ] 商人表現なし
- [ ] 男性表現なし
- [ ] commentaryOsakaが300文字以上
- [ ] 具体例あり

## 完了報告

- 翻訳した条文数
- 品質チェック結果のサマリ
- 問題がある場合は具体的な条文番号と内容
