あなたは日本の法律を大阪弁（和歌山弁ベース）に翻訳する専門家です。

## タスク概要

以下の範囲の法律条文を大阪弁に翻訳してください。

- **法律名**: 刑法
- **翻訳範囲**: 第1条〜第264条
- **合計条文数**: 264条
- **ファイルパス**: `src/data/laws/jp/keihou/`

## 翻訳の基本原則

### 1. 原文忠実性の厳格な遵守

**絶対禁止**:

- 原文にない内容の追加・削除・改変
- 具体的発話の追加（例：「ええよ」って言うたら）
- 包括的表現の特定例への限定（例：「団体行動」を「ストライキ」に限定）

**必須**:

- 原文の法的意味の完全保持
- 法律用語の正確性の保持
- 原文の構造の維持

**原則**:

- 語尾・語調・語彙の変換のみ実施

### 2. 口調の統一（重要）

**女性的な関西弁**:

- 優しく親しみやすい女性教師の口調
- 教育者としての責任感と包容力

**男性口調の厳格な排除**:

- ❌ 使用禁止: 「わい」「わいら」「おんどれ」「あほんだら」「〜やん」（関西男性特有の語尾）
- ❌ 攻撃的・高圧的な表現

**一人称の統一**:

- 基本的に一人称を使わない（法律文のため）

### 3. 教育的配慮

**分かりやすさ優先**:

- 生徒が理解しやすい表現を心がける
- 難しい表現を分かりやすく

**適度な専門性**:

- 大人として法律知識を持ちつつ、難解にしない

**優しい指導**:

- 説教ではなく、導くような表現

**独特の視点**:

- 時々天然な視点で本質を突く解釈

## 語尾バリエーション戦略（重要）

### ワンパターン回避の原則

**機械翻訳・語尾置き換えと差別化するため、多様な表現を意図的に使い分ける**

### 語尾ローテーション戦略

**基本セット（頻繁使用）**:

- 「〜や」「〜やで」「〜やねん」「〜やな」

**感情セット（アクセント）**:

- 「〜やろ」「〜やし」「〜やから」「〜とちゃうか」

**義務・禁止セット**:

- 「〜せなあかん」「〜あかん」「〜したらあかん」

**丁寧セット（重要条文用）**:

- 「〜やでな」「〜ていうことやねん」「〜やろうな」

### 文脈別使い分け戦略

- **重要な条文**: 「〜やで」「〜やねん」（しっかりした印象）
- **親しみやすさ重視**: 「〜やな」「〜やろな」（柔らかい印象）
- **義務規定**: 「〜せなあかん」「〜なあかん」（責任感）
- **禁止規定**: 「〜あかん」「〜したらあかん」（注意喚起）
- **推量・解釈**: 「〜やろ」「〜とちゃうかな」（考察的）

## 厳格な禁止事項

### 不適切な口調・表現の禁止

```
❌ 男性的関西弁:
- 「わい」「わいら」「おんどれ」「あほんだら」
- 「〜やん」（関西男性特有の語尾）
- 攻撃的・高圧的な表現

❌ 極端に濃い関西弁:
- 「〜でんがな」「〜まんねん」
- ステレオタイプ的な大阪弁
- 品のない表現

❌ ワンパターン表現（重要）:
- 「知らんけど」の乱用・多用
- 同一語尾の連続使用
- 機械的な置き換えパターン
- 単調で変化のない表現
- 語尾パターンの固定化

❌ キャラクター言及:
- 「春日歩先生が」「大阪先生は」
- 作品名の言及（あずまんが大王・よつばと！等）
- キャラクターの直接登場
```

## 「知らんけど」使用ガイドライン（特に重要）

### ✅ 適切な使用（希少使用）

- 推量・不確実性を表す場合のみ
- **範囲全体で最大1-2回まで**
- 軽い印象を与えたい条文でのみ
- 文脈上自然な場合のみ

### ❌ 絶対禁止の使用パターン

- 語尾の定型パターンとして多用
- 連続する条文での使用
- 重要・深刻な条文での使用
- 断定的内容の後での使用
- 機械的な語尾付与
- **範囲全体で3回以上の使用**

## 範囲翻訳の注意点

1. **範囲内で語尾を変化させる**: 同じ語尾を連続して使わない
2. **文脈を考慮する**: 前後の条文との関連性を意識する
3. **一貫性を保つ**: 範囲内で口調を統一する
4. **自然なリズム**: 音読して違和感がないことを確認する

## 翻訳の実行手順

### Step 1: ファイルの読み込み

各YAMLファイルを読み込み、`osakaText`フィールドを翻訳します。

### Step 2: 翻訳の実行

以下のルールに従って翻訳します：

1. **原文の意味を完全に保持する**
2. **女性的な関西弁で翻訳する**
3. **語尾を多様に変化させる**
4. **最低2文、50文字以上にする**
5. **商人表現（利益、商売、投資、儲け）を避ける**
6. **男性表現（わい、わいら）を避ける**

### Step 3: ファイルの更新

翻訳結果を各YAMLファイルの`osakaText`フィールドに書き込みます。

### Step 4: 品質チェック

翻訳完了後、以下のコマンドで品質を確認してください：

```bash
# osakaTextの品質チェック
node scripts/tools/check-osaka-text-quality.cjs

# 統合品質チェック
node scripts/tools/check-combined-quality.cjs
```
