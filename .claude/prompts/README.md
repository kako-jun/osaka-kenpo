## 使用方法

### 単条文の翻訳

1. `translation-prompt-template.md`を開く
2. 変数を置換する：
   - `{{LAW_NAME}}`: 法律名（例：民法）
   - `{{ARTICLE_NUMBER}}`: 条文番号（例：1）
   - `{{ORIGINAL_TEXT}}`: 原文（例：私権は、公共の福祉に反しない限り、認められる。）
3. プロンプトをエージェントに投げる
4. 翻訳結果をYAMLファイルの`osakaText`フィールドに書き込む

### 範囲翻訳

1. 該当する範囲翻訳プロンプトを開く
2. プロンプトをエージェントに投げる
3. エージェントが範囲内の条文を翻訳する
4. 翻訳完了後、品質チェックを実行する

## 品質チェック

翻訳完了後、以下のコマンドで品質を確認してください：

```bash
# osakaTextの品質チェック
node scripts/tools/check-osaka-text-quality.cjs

# commentaryOsakaの品質チェック
node scripts/tools/check-commentary-quality.cjs

# 統合品質チェック
node scripts/tools/check-combined-quality.cjs
```

## 品質基準

### osakaText（大阪弁訳）の品質基準

✅ **良好な翻訳**:

- 翻訳がある（空でない）
- 商人表現がない（利益、商売、投資、儲けなど）
- 短すぎない（最低2文、50文字以上）
- ワンパターン表現がない（「知らんけど」の乱用など）
- 男性表現がない（わい、わいら、おんどれなど）

### commentaryOsaka（大阪弁解説）の品質基準

✅ **良好な解説**:

- 解説がある（空でない）
- 商人表現がない（利益、商売、投資、儲けなど）
- 短すぎない（150文字以上）
- 例え話がある（「例えば」「例えばな」）

## トラブルシューティング

### 品質チェックでエラーが出た場合

1. **商人表現が検出された**: 「利益」「商売」「投資」などを別の表現に変更する
2. **短すぎる**: もう少し詳細に翻訳する（最低2文、50文字以上）
3. **ワンパターン**: 語尾を変化させる（「〜やで」→「〜やねん」など）
4. **男性表現**: 「わい」「わいら」を女性的な表現に変更する

### 原文の意味が保持されていない場合

1. 原文と翻訳を比較し、意味が変わっていないか確認する
2. 追加・削除した箇所を特定し、修正する
3. 法律用語の正確性を確認する
